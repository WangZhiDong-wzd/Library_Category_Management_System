🔄 数据流与存储策略
用户操作 → jQuery 事件 → 业务逻辑 → CRUD 操作 → Web Storage → DOM 刷新
数据类型存储方式生命周期说明
📚 图书类别 sessionStorage 会话级 关闭标签页自动清除，避免错误配置持久化
🏅 用户等级 sessionStorage 会话级 系统配置类数据，每次会话使用默认值重置
📦 订单数据 localStorage 持久化 核心业务数据，跨浏览器会话保留
👤 用户数据 localStorage 持久化 用户账号信息，持久保留不丢失

✨ 技术亮点
  • 1
    分层数据持久化策略 — 配置类数据（类别 / 等级）使用 sessionStorage，避免修改后持久化污染；业务数据（订单 / 用户）使用 localStorage，确保关闭浏览器后数据不丢失。
  • 2
    SPA 页面架构 — 使用 iframe + 手风琴菜单实现单页应用效果，模块独立开发互不干扰，复用 Bootstrap 布局体系。
  • 3
    数据可视化 — 基于 Chart.js 实现柱状图（类别销量对比）和折线图（7 日订单趋势），差异化配色 + 曲线平滑 + 入场动画，提升数据可读性。
  • 4
    无障碍（ARIA）支持 — 菜单区域标注 role="menubar/menuitem"，标签页使用 role="tablist/tab"，表单错误使用 role="alert" 实时播报，提升辅助技术兼容性。
  • 5
    模块化 CSS 隔离 — 每个功能页面拥有独立 CSS 文件，避免样式污染；登录页采用 CSS 变量 + 暖色书香主题，仪表盘卡片 hover 上浮动效。
  • 6
    完整的表单校验体系 — 非空检查 · 手机号正则 · 密码最小长度 · 邮箱格式 · 用户名唯一性校验 · 实时输入清除错误状态 · 聚焦首个错误字段。
